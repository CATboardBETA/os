# Contributing

Thank you for wanting to contribute! 

First of all, and this is very important—**AI assisted-code or AI-generated code will not be tolerated**. This project
has been written entirely by humans, and I intend to keep it that way. This rule is **non-negotiable**. Any issues in
violation of this rule will be closed without discussion, and PRs openly admitting to having used AI will have the same.

Maintainers reserve the right to deny PRs for any reason, including suspected AI usage, poor code practices, rudeness,
or not following code style.

Please create an issue before implementing any new features or fixing any bugs which take a lot of work, to prevent 
wasted time.

Try to keep PRs isolated to just fixing one bug, or adding one new feature. This isn't a huge deal in the early stages
(it won't be enforced whatsoever), but when things are slightly more stable I'd *also* like them to be more organized.

Before submitting your PR, I'd recommend running `cargo fmt` (if you choose not to, a reviewer will request you do 
before your PR is accepted). Also take a look at STYLE.md.

Lastly, make sure when you submit your PR you have added your name to the CONTRIBUTORS.md file for recognition. No 
addition is too small.
